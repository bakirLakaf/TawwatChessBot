# دليل تشغيل بوت Tawwat Chess — الكامل

بوت واحد يدير محتوى صفحتك:
- **الأخبار:** يقرأ مصادر RSS، يصيغها بالعربية بأسلوبك، ويرسلها لك على تيليجرام للموافقة قبل النشر.
- **محتوى مجدول في أوقات ثابتة:** ألغاز (من Lichess) · حكم شطرنجية · نقلات من الزمن الجميل (مباريات أسطورية مُتحقَّقة) · إعلان البطولة الأسبوعية.
- حل اللغز يُوضع تلقائيًا في **أول تعليق**.

---

## ١) كيف يعمل
- **الأخبار:** كل ساعتين (افتراضيًا) → جديد فقط → صياغة عربية (مجاني أو Claude، بلا اختراع) → بطاقة بقالبك → تيليجرام (أزرار: نشر/تعديل/حذف) → فيسبوك بعد موافقتك.
- **المجدول:** في وقته (انظر الجدول) يجهّز المحتوى، ثم:
  - `SCHEDULED_AUTO_PUBLISH=false` → يصلك للموافقة في وقته (آمن، مستحسن للبداية).
  - `SCHEDULED_AUTO_PUBLISH=true` → نشر تلقائي هادئ في وقته (هاندز-فري).

### الجدول الأسبوعي الافتراضي (توقيت الجزائر)
| اليوم | المحتوى | الوقت |
|------|---------|-------|
| الأحد | نقلة من الزمن الجميل | 21:00 |
| الإثنين | حكمة شطرنجية | 13:00 |
| الثلاثاء | لغز | 21:00 |
| الأربعاء | حكمة شطرنجية | 13:00 |
| الخميس | نقلة من الزمن الجميل | 21:00 |
| الجمعة | إعلان البطولة | 21:00 |

عدّله بسهولة في `config.py` (قائمة `SCHEDULE`). الأيام: 0=الأحد … 6=السبت.

## ٢) المفاتيح المطلوبة (ملف `.env`)
انسخ `.env.example` إلى `.env` واملأه:
- **تيليجرام:** `TELEGRAM_BOT_TOKEN` من `@BotFather` · `TELEGRAM_ADMIN_CHAT_ID` من `@userinfobot`. واضغط **Start** في محادثة بوتك.
- **محرّك صياغة الأخبار (اختر واحدًا):**
  - *مجاني (مستحسن):* مزوّد متوافق مع OpenAI. املأ `LLM_API_BASE` و`LLM_API_KEY` و`LLM_MODEL`.
    - **Gemini المجاني** (الأكثر موثوقية): مفتاح مجاني من `aistudio.google.com` → `LLM_API_BASE=https://generativelanguage.googleapis.com/v1beta/openai/` و`LLM_MODEL=gemini-2.5-flash`.
    - **bynara**: مفتاح من `router.bynara.id` → `LLM_API_BASE=https://router.bynara.id/v1` و`LLM_MODEL=mimo-v2.5-free`.
  - *مدفوع:* `ANTHROPIC_API_KEY` من `console.anthropic.com` (اتركه فارغًا إن استعملت المجاني).
  - ملاحظة: الأخبار فقط تحتاج هذا المحرّك؛ الألغاز/الحكم/النقلات تعمل بدونه.
- **فيسبوك:** `FB_PAGE_ID` و `FB_PAGE_ACCESS_TOKEN` (توكن الصفحة طويل الأمد، بصلاحيتَي `pages_manage_posts` و `pages_read_engagement`).

## ٣) التشغيل محليًا (ويندوز)
1. ثبّت Python 3.10+.  2. جهّز `.env`.  3. شغّل `start_bot.bat`.
4. في تيليجرام: `/start` ثم جرّب الأوامر.

### الأزرار والأوامر
بعد `/start` تظهر **لوحة أزرار دائمة** أسفل المحادثة (لا حاجة لكتابة الأوامر):
📰 فحص الأخبار · 🧩 لغز · 💡 حكمة · 📜 نقلة · 🏆 بطولة · ⚙️ لوحة التحكم.

نفس الأوامر بالكتابة: `/check` · `/puzzle` · `/wisdom` · `/classic` · `/tournament` · `/control`.

**⚙️ لوحة التحكم** فيها:
- ⏸️/▶️ إيقاف/تشغيل الأخبار التلقائية.
- 📊 إحصائيات (ملتقطة/منشورة/بانتظار/محذوفة/فاشلة).
- 🔄 تحديث قوائم اللاعبين بعد تعديل ملفات JSON (بلا إعادة تشغيل).

> الأوامر اليدوية تُرسل المحتوى **للموافقة دائمًا** (مناسبة للتجربة الآمنة).

### الأخبار بنسختين (عربي + إنجليزي)
لكل خبر يصلك بطاقة واحدة بأزرار:
- **✅ نشر** — ينشر النسخة المعروضة حاليًا.
- **🇬🇧 English / 🇸🇦 العربية** — يبدّل المعاينة بين اللغتين (تقارن وتختار).
- **✏️ تعديل النص** — يستبدل نص النسخة المعروضة.
- **🖼️ الصورة** — اضغطه ثم أرسل **صورة** (أو **رابطًا** يبدأ بـ http) لتغيير صورة البطاقة (للنسختين). مفيد حين تضع المصادر صورة عامة لا تخصّ الحدث.

### هوية البطاقة حسب الموضوع
- **ترويسة البطاقة** تعرض **اسم البطولة/الحدث** (مثل `Norway Chess 2026`) بدل «أخبار الشطرنج»، وتتقلّص تلقائيًا لتتّسع.
- **اللون ونمط الخلفية والشارة** يتغيّران حسب تصنيف الخبر: نتائج/بطولة/ذكريات/حوار/افتتاح. (أسماء اللاعبين والبطولات تبقى بحروفها اللاتينية.)
- **تنبيه عن الصور:** تأتي الصورة من الموقع المصدر؛ أحيانًا يضع صورة عامة لا تطابق اللاعب/الحدث — حينها استعمل زر **🖼️ الصورة** لتبديلها قبل النشر. (يتجاهل البوت الشعارات/البانرات الواضحة تلقائيًا.)

## ٤) النشر على Railway (24/7)
1. ارفع المشروع إلى GitHub خاص (بدون `.env`).
2. railway.app → Deploy from GitHub → اختر المستودع.
3. Variables: أضف كل المفاتيح (**لا تضع `PORT`** — Railway يحدّده).
4. Railway يقرأ `Procfile` ويشغّل البوت تلقائيًا.

## ٥) التخصيص
- **الجدول:** `SCHEDULE` في `config.py`.
- **مصادر الأخبار:** `FEEDS` في `news_sources.py` (Chess.com مؤكّد؛ تحقّق من البقية).
- **الحكم:** أضف مدخلات في `data/quotes.json` (الصيغة: `{"q": "النص", "a": "القائل"}`).
- **نقلات الزمن الجميل:** أضف في `data/classic_games.json`. كل مدخل يحتاج `fen` (وضعية صحيحة) و `story`. خذ الـFEN من Lichess/Chess.com.
- **شكل البطاقة:** `template.py` (الهوية الحالية المعتمدة).

## ٥-أ) التركيز على أبرز اللاعبين في الأخبار
البوت يعطي **الأولوية** للأخبار التي تذكر لاعبين بارزين، دون أن يحذف بقية الأخبار:
- **عالميًا:** `data/top_players.json` (مبنيّ على تصنيف FIDE الكلاسيكي).
- **عربيًا:** `data/arab_players.json` (يُبرَزون بأعلى أولوية + وسم `#لاعبون_عرب`).

الترتيب في كل دورة فحص: **لاعب عربي ← لاعب عالمي ← خبر عام**. الأخبار الأقلّ أولوية لا تُفقَد، تبقى مرشّحة للدورات التالية.

**لإضافة/حذف لاعب:** عدّل ملفّ JSON المناسب فقط — كل عنصر:
```json
{ "name": "الاسم المعروض", "aliases": ["Surname", "الاسم بالعربية", "تهجئة بديلة"] }
```
- ضع في `aliases` كل التهجئات التي قد ترد في الأخبار (إنجليزي + عربي). الأخبار غالبًا إنجليزية، فالاسم/اللقب الإنجليزي هو الأهمّ للمطابقة.
- تجنّب الألقاب القصيرة جدًا أو الكلمات الشائعة وحدها (مثل `Arab` أو `So`) حتى لا تحدث مطابقات خاطئة — استعمل الاسم الكامل لها.
- لا حاجة لتعديل الكود. (القوائم تُحمَّل عند الإقلاع.)

## ٥-ب) تنبيهات الأخطاء على تيليجرام
أي خطأ أثناء التنفيذ (فشل جلب خبر، فشل صياغة، فشل نشر على فيسبوك، تعطّل غير متوقّع…) يصلك تلقائيًا على محادثتك في تيليجرام. لا إعداد إضافي — يستعمل نفس توكن البوت. عند الإقلاع تصلك رسالة «البوت اشتغل ✅».

---

## ٦) حل المشاكل
- **البوت لا يراسلني:** لم تضغط Start في محادثة البوت، أو الـchat id خاطئ.
- **فشل النشر على فيسبوك:** التوكن منتهٍ/ناقص الصلاحيات → جدّده.
- **«تعذّر تجهيز اللغز»:** خدمة Lichess غير متاحة لحظيًا → جرّب لاحقًا.
- **حروف عربية مفكّكة على ويندوز:** `pip install -U Pillow`.
- **بطاقة الخبر بلا صورة (الأيقونة الافتراضية):** يجلب البوت صورة المقال من صفحته تلقائيًا (`og:image`) حين لا تضعها خلاصة RSS؛ إن غابت من المصدر أصلًا تظهر الأيقونة الافتراضية.

- توسيع قائمتَي اللاعبين تلقائيًا من تصنيف FIDE.
